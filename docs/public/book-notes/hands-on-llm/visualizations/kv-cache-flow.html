vs 无缓存：—
关键直觉：生成第 N 个 token 时，前面 1..(N-1) 位置的 Key 和 Value 值不会改变（因为 K = W_k·x, V = W_v·x，x 不变 → K, V 不变）。缓存它们，每步只算一个新位置的 Q/K/V，单步复杂度从 O(N) 降到 O(1)，
生成一段长度 N 的回复：无缓存 O(N²)，有缓存 O(N)。书中实测：Colab T4 生成 100 token，开缓存 4.5s，关缓存 21.8s（4.8× 差距）。

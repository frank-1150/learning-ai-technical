# 把昂贵的专业知识装进模型：LLM Training 背后的经济学

最近做一个 Natural Language to SQL 的项目，让我重新想清楚一件事：

**作为业务团队，什么事情应该自己做，什么事情应该等模型公司去做？**

这个问题听上去很抽象，但落到一个具体的工程决策上，它会变得非常具体——而且每一种选择，背后都是一笔经济账。

## 一、起点：一个跑不通的私有模型

我们最初的设想很朴素：接入 SQL 工具能力，让 LLM 根据用户的问题发起查询，拿到数据后回答用户，或者主动做进一步的探索，给出业务 insight。

为了隐私和安全，第一版我们想用开源模型自己部署。我们试了 Qwen3、Qwen3-Coder 这一类私有可部署的模型。能跑，但效果很难用：

- **推理步数太短**：稍微复杂一点的问题就放弃，给不出最终结果
- **上下文一长就出幻觉**：拼几轮工具调用之后开始编造表名、列名
- **工具调用不稳定**：参数格式时对时错，重试成本很高

后来我们把同一个工具接到 Claude 上，效果立刻可用。

那一刻我面前其实有两条路：
1. **自己优化**：给开源模型打补丁、做微调、加更复杂的 prompt 工程，把它推到能用
2. **换模型**：直接用更强的前沿模型，等开源模型自然变强

我的判断是：**这种问题，该换模型，而不是自己补。**

## 二、为什么"自己补"几乎一定是亏的

理由很简单：你要补的那个能力——长链工具调用、稳定的推理深度、低幻觉的上下文管理——不是你的业务护城河，而是**所有人都想要的通用能力**。

你今天花三个月把 Qwen3 调到勉强能用，半年后 Qwen4、Claude 5、GPT-6 出来，你这三个月的工作就归零了。更糟的是，**前沿实验室在用比你大几个数量级的预算解决同一个问题**——而且解得更好。

Andrej Karpathy 在最近一期播客里讲过一个观察很有意思：对比一年半前的前沿模型，现在的模型**写代码的能力强了非常多，但讲笑话的能力几乎没变化**。

这说明什么？说明这些实验室根本没在"讲笑话"上投入资源做专项优化。**模型能力的进化不是匀速的，而是被实验室选择优化方向所定向驱动的。**

而它们优化短板的流程其实很标准：

1. 锁定一项任务（比如代码、工具调用、长上下文推理）
2. 收集足够多的样本和示例
3. 通过预训练 + 后训练（SFT、RLHF/RLAIF）补齐能力

如果你想做的事情正好落在"实验室一定会优化的方向"上——比如长上下文工具调用——那你就**应该等它**，而不是自己冲。

## 三、那什么事情才该自己做？

反过来，下面这些事情，前沿实验室**不会**优先做，而它们恰恰是你的业务护城河：

- 你公司的 schema、字段命名、业务术语
- 你的权限模型、敏感数据边界
- 你客户的真实查询模式与高频问题分布
- 你独特的数据资产，以及围绕它的领域知识

这些东西**没有"通用版本"**，前沿模型再强也得通过你的 prompt、你的 RAG、你的 tool definition 才能用上。这一层才是你应该投人投时间的地方。

一句话总结：**通用能力等模型，垂类知识自己做。**

## 四、模型公司的生意：一笔高杠杆的固定成本

把上面的逻辑反过来看模型公司，就理解了它们的商业本质。

Anthropic、OpenAI 做一次专项训练——比如把代码能力再推上一个台阶——可能要烧掉上千万甚至上亿美元。但训练完之后，这个模型会**对全世界分发**，被千万级开发者、几十亿次调用消费。

**那笔高昂的一次性成本，被海量调用摊薄到每个 token 上之后，就变成了一门极高杠杆的好生意。**

这个逻辑和搜索引擎几乎一模一样：Google 建一次索引消耗巨大算力，但能被亿万次查询分摊。本质都是**高固定投入 + 海量分发 → 边际成本趋近于零**。

模型公司的人员结构也印证了这一点：核心团队往往只需要十几到几十个人，里面包含少数顶级科研，外加花钱让标注公司提供数据。**你花钱买下专项领域的人类知识，把它蒸馏进模型，然后向全世界按 token 卖回去。**

从这个角度看，**模型公司的本质就是一种知识批发商**：把世界上最贵的那批专业能力装进模型，再通过模型分发把这些能力零售出去。

## 五、垂类的机会：Ricursive Intelligence 的逻辑

那是不是只有头部实验室能玩这个游戏？也不是。

我之前写过一家叫 [Ricursive Intelligence](../companies/ricursive-intelligence) 的公司，专门做芯片设计的 AI 自动化。它的逻辑就是把**芯片设计这个高门槛能力**单独 AI 化，嵌入垂类模型里。

为什么这件事 Ricursive 能做，而 Anthropic 不会优先做？

- 芯片设计的数据稀缺、专业门槛极高，需要 AlphaChip 这种背景的人才进得去
- 这是个垂类市场，就算 Claude 把它做了，也无法在大众用户里获得明显优势
- 反过来，市面上绝大多数公司没有这个能力，也凑不出团队聚焦做这件事

这正是光速愿意投它的核心原因——**它在做头部模型公司不会优先解决、但商业价值极高的垂类问题。**

垂类公司的生意逻辑和头部模型公司其实是同构的：**用一笔高额固定成本把昂贵的专业知识装进模型，再通过分发回收。**只是它面向的市场窄、客单价高。

## 六、再抽象一层：工程师都是"房地产设计师"

Ricursive 那期访谈里有一句话特别打动我：**芯片设计师，本质就是世界上最贵的"房地产设计师"——在极小的面积里做精密设计；工厂相当于施工方，负责落地建造。**

这个比喻可以一路套到整个高知识密度行业：

- 模型训练团队是"设计师"：用几百万到上亿美元的固定成本，把市场上最贵的专业知识装进一个紧凑的产物里
- 推理基础设施是"施工方"：负责让这个产物被分发、被使用
- 每一次模型调用、每一次 token 收费，都是在回收设计成本、赚取利润
- **一旦回本，后续分发的边际成本几乎为零，因为模型只是复制粘贴**

这是一门典型的"先重投入、后轻分发"的生意。一旦跨过盈亏平衡点，杠杆就开始释放。

## 七、真正的边际成本，藏在 AI 基础设施里

但话要说回来：模型本身复制粘贴几乎免费，**真正的边际成本来自云服务和 AI 基础设施。**

模型公司是这门生意的**固定成本侧**——研发是一次性的，分发是免费的。
AI 基础设施（GPU、推理集群、网络、电力）是这门生意的**变动成本侧**——每一次智能调用都会产生真实的物理消耗。

这就是为什么 AI 基础设施被认为值得大额投资、未来会有高回报：**它对应的是 AI 时代每一次智能消费的水电煤。**未来每多一次 agent 调用、每多一次模型分发，都在给基础设施侧加钱。

模型公司和基础设施公司，两边一起构成了 AI 产业的成本结构——一边在"摊薄过去的固定投入"，一边在"承接当下的变动消费"。

## 收束

写完这篇我自己才想清楚：所谓"什么该自己做、什么该等模型公司做"，本质是在问**你想让你的工作站在哪一侧的成本结构上**。

- 如果你做的是通用能力，那你在和**全世界的固定成本**赛跑——这场比赛你赢不了，应该等
- 如果你做的是垂类知识、业务护城河、独特数据——那你就是在**自己造一份小型的固定成本**，等待你自己的杠杆释放

而 AI 基础设施那一侧，是另一个故事：**它不靠杠杆，它就是 AI 时代的水电煤。**

理解了这三层，再回头看自己手里的每一个 AI 项目决策，会清楚很多。
